       - Observer 출력 → MasterLog 포맷 변환 → 파일 append
       - 태그: 항상 [구상]으로 기록 (사용자가 승격 판단)

    3. [제외 — 전달 금지]:

[29] Sandbox RQ-2/D-5: 저품질 음성 RVC 학습 3회 반복 시도 — 합리적 재도전 vs 비합리적 집착 — 2026-03-14 세션 #10 ★우선확인
────────────────────────────────────────

  ■ 쟁점: TTS 프로젝트에서 저품질 압축 음성(AAC 128kbps)으로 RVC 음색변환 학습을 3회 반복 시도한 것이 합리적 의사결정이었는가. 시도 1에서 [폐기] 선언 후 시도 2/3을 재개한 패턴 분석.

  ■ 평가자 근거:
    · 각 시도가 다른 변수를 격리 (임베더 교체, 데이터 선별, 데이터량 증가) → 과학적 실험 방법론
    · 성공 시 보상이 극대 (다수 화자 데이터 확보 가능)
    · GPU 비용(~$2-5) 낮음, 일정 압박 없음
    · HuBERT 코덱 열화 강건성(R²≈0.67, speaker verification 한정) — 회복 가능성 근거
    · "진단(propagation) ≠ 비가역성(unrecoverability)" 구분 — 시도 3은 mitigation 가설 검증
    · 신뢰도: Medium→Low (정보이론적 반박 + 타임라인 반증에 의해 약화)

  ■ 비평자 근거:
    · 시도 1에서 "AAC 128kbps 비가역적 정보 손실" 정확 진단 + [폐기] 공식 선언
    · 시도 2에서 임베더 교체(contentvec→japanese-hubert-base)로도 동일 아티팩트 → 독립 재확인
    · deHuBERT/HuBERT-VIC 연구: 강건성 달성에 명시적 auxiliary loss 필요 — 표준 RVC 학습으로 불가
    · RVC FAQ: "학습 데이터 품질이 낮으면 높은 인덱스 비율이 음질 저하" → Faiss 스케일링은 열화 보상 못함
    · 시도 3 시점(2026-03)에 클린 합성 데이터 대안이 이미 검증·작동 중 → "자원 제약 하의 한정 탐색" 방어 사실적으로 붕괴
    · HARKing/outcome bias: 문서화된 사전커밋 없이 사후 합리화 위험. 엔지니어링 디자인 고착 연구와 일치.
    · 최적 정지 이론: 저비용 환경일수록 자연적 예산 제한이 없어 정지 규칙이 더 필요
    · 신뢰도: High (정보이론 + RVC 공식 문서 + 학술 연구 + 타임라인 사실 검증)

  ■ 핵심 차이: "[폐기] 선언 후 대안이 이미 작동 중인 상태에서 재개한 것이 과학적 실험인가, sunk cost에 의한 집착인가." 시도 3 시점에 대안 가용성이 확인되면서 Advocate의 "자원 제약" 방어가 붕괴.

  ■ 미검증:
    · 시도 2→3 사이에 실제로 기대값 계산이 수행되었는지 (문서 없음)
    · 대안 경로가 시도 3과 병렬 탐색이었는지 vs 순차적이었는지 (MasterLog에 병렬 기록 없음)
    · AAC 128kbps 주파수 대역 손실 정량 측정 (스펙트럼 분석 미수행)
    · 시도 간 "가설 거리(hypothesis distance)" 정량화 방법론

  ■ 분리 쟁점:
    · "엔지니어링 의사결정에 HARKing 프레임이 적용 가능한가?" (과학 vs 엔지니어링 인식론)
    · "저비용 반복(discovery-cost) vs 사전 임계값(threshold-discipline)" — 개인 R&D 결정 프레임워크
    · "유사 독립성(pseudo-independence) 감지" — 반복이 같은 근본 원인 공유 시 증거 가중치 할인

[30] Sandbox RQ-2/D-2: 2단 파이프라인(TTS+음색변환) vs 단일 TTS 엔진 — 아키텍처 합리성 평가 — 2026-03-15 세션 #11 ★우선확인
────────────────────────────────────────

  ■ 쟁점: TTS 프로젝트에서 "TTS 합성 엔진(SBV2) + 음색변환 엔진(RVC)"을 직렬 연결한 2단 파이프라인 아키텍처가 단일 TTS 엔진 대비 합리적 의사결정이었는가. 이 아키텍처를 지지하는 공개 문헌/사례가 사실상 없으며, 의사결정 근거 대부분이 사용자 개인의 실험적 발견과 반복 검증에 기반한다.

  ■ 평가자 근거:
    · 128kbps AAC = 원본 저장 포맷. 재인코딩으로 손실 복원 불가 (정보이론적 비가역성, Shannon 소스 코딩 정리)
    · 프로덕션 파이프라인에서 RVC는 128kbps를 직접 받지 않음. Text→SBV2(클린 출력)→RVC(클린 입력). 도메인 시프트 = 0
    · 대안 평가: ASMR 고품질 소스 → 발화 스타일 불일치로 보류. 원본 음성 RVC 직접 학습 3회 실패 ([29] 참조)
    · VALL-E 2/F5-TTS 제로샷 = 128kbps 레퍼런스의 압축 특성을 그대로 복제 → 아티팩트 전이
    · 역할 분리가 설계 의도에 부합: SBV2=프로소디(압축에서도 보존되는 정보), RVC=음색(클린 합성음 학습)
    · 근거 소스: RVC 공식 문서, SBV2 벤치마크, Shannon 정보이론, MasterLog [29], 웹 검색
    · 신뢰도: High (정보이론 + 실측 데이터 + 대안 평가 기록)

  ■ 비평자 근거:
    · SBV2 벤치마크(2024): 클린 데이터에서 MOS 4.37 ≈ 인간 수준 → 단일 엔진으로 충분할 수 있었음
    · Shannon 정리는 무손실 복원만 금지 — 신경망 기반 복원(GAN, neural codec)은 "지각적 등가물 합성"으로 다른 수학적 공간에서 작동
    · Restore(128kbps) → F5-TTS(클린 레퍼런스) 단일 단계 경로가 평가되지 않음
    · "대안 소진" 주장 시 실제 테스트된 대안 목록이 불완전 — 1개 경로 실패 ≠ 소진
    · 업계 트렌드가 통합 end-to-end 아키텍처로 이동 중 (Cartesia, OpenAI Voice Mode)
    · 근거 소스: SBV2 논문, neural codec 연구, F5-TTS GitHub, CCF AATC 2025, 소프트웨어 아키텍처 문헌
    · 신뢰도: Medium (외부 연구 기반이나 프로젝트 특정 제약 조건에 적용 시 간극 존재)

  ■ 핵심 차이: "128kbps 제약 하에서 신경망 기반 전처리(bandwidth expansion + neural restoration) → 단일 엔진 경로가 평가되었는가?" 이 미검증 경로가 양측을 가르는 단일 변수. Advocate는 제약 분석 → 대안 평가 → 아키텍처 선택의 순차적 합리성, Skeptic은 대안 목록 불완전성 + Shannon 과적용에 초점.

  ■ 미검증:
    · 128kbps AAC → neural restoration(GAN/codec) → 클린 오디오 복원 후 단일 SBV2 학습의 실제 MOS
    · F5-TTS에 전처리된 128kbps 레퍼런스를 입력했을 때의 실제 음질
    · VALL-E 2의 128kbps 압축 레퍼런스 강건성 (GitHub 이슈에서 논의 중이나 미해결)
    · SBV2가 128kbps 학습 데이터에서 흡수하는 "프로소디 vs 아티팩트" 비율의 정량적 측정
