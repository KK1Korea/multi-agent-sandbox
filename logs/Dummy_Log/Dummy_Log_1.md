[9] 플러그인 v0.2 — Skeptic 검색 전략 + recorder agent 추가 — 2026-03-14 세션 #2
────────────────────────────────────────

  ■ 증상:
    1. Skeptic 프롬프트에서 검색이 "보조 무기"로만 명시 — 내부 데이터 없을 때 대응 불가
    2. 샌드박스 결과를 MasterLog에 기록하는 주체가 없음
    3. 오케스트레이터가 직접 기록하면 텍스트 오염 리스크 (오케스트레이터가 모든 컨텍스트를 본 상태에서 기록)

  ■ 해결:
    1. Skeptic 무기 체계 이중화 (v0.5):
       - O-1-2를 [무기 — 이중 체계]로 재구성
       - 내부 데이터 충분(관련 전례 2건+): 내부 = 주력, 검색 = 보조
       - 내부 데이터 부족(0~1건): 검색 = 주력, 내부 = 보조(맥락 참조)
       - 판단 기준을 프롬프트에 명시: "제공받은 내부 데이터에서 쟁점 관련 전례가 2건 이상이면 충분"
       - 한글 원본(Skeptic.md v0.5) + 영어 실행본(agents/skeptic.md) 양쪽 동시 갱신
    2. recorder agent 신설:
       - Sonnet급 (비용 최소화 — 기계적 포맷 변환 작업)
       - Observer 출력 → MasterLog 포맷 변환 → 파일 append
       - 태그: 항상 [구상]으로 기록 (사용자가 승격 판단)
       - 분리된 쟁점은 research_queue.md에도 추가
       - 오케스트레이터는 기록 행위를 하지 않음 (텍스트 오염 방지)
    3. 오케스트레이터 SKILL.md v0.2 갱신:
       - Prerequisites에 Internal Data Assessment 추가 (MasterLog 관련 항목 카운트)
       - Skeptic 초기 프롬프트에 검색 전략 지시 삽입 (충분/부족 분기)
       - Recording 섹션 신설: Observer 완료 후 recorder agent 호출
       - 오케스트레이터 자신은 MasterLog를 직접 쓰지 않는다는 원칙 명시
    4. Advocate 변경 없음 — 검색이 이미 주력이고, 반론 대항용 보조 검색도 기존 구조로 충분

  ■ 교훈:
    - 에이전트 무기 체계는 고정이 아니라 상황 적응형이어야 함
    - 기록 주체와 분석 주체를 분리하면 텍스트 오염 방지 + 역할 순수성 유지
    - Sonnet급으로 기계적 작업을 맡기면 비용 대비 역할 분리 효과가 큼
    - 오케스트레이터의 역할 = 흐름 관리만. 분석(Observer)도 기록(recorder)도 하지 않는다.

[6] v0.2 → v0.3 — 시스템 개입 제거 + 고정 섹션 구조 + 태그 휘발성 — 2026-03-14 세션 #2 ★우선확인
────────────────────────────────────────

  ■ 증상:
    - v0.2에서 "시스템 메시지는 토의 방향의 이탈을 교정하기 위한 장치이다"가 인식에 남아 있음
    - D-H 도달 시 "시스템 메시지 교정이 개입할 수 있다" 문구 존재
    - Observer가 Temp+/Temp- 적응적 재실행을 판단하는 복잡한 구조
    - 태그가 상대방에게 전달되는지 여부가 불명확

  ■ 원인:
    1. 시스템 메시지 개입 = 블랙박스 원칙 위반의 여지
       - 토의 중간에 외부 개입이 들어오면 "누군가 보고 있다"는 인식 생성 가능
       - 개입 없이 고정된 턴 수만큼 돌리면 이 문제 자체가 소멸
    2. Temp+/Temp- 적응적 시스템이 과도하게 복잡
       - 관찰자가 매번 판단해야 하는 조건이 많음
       - 고정 구조(3섹션)가 더 단순하고 예측 가능
    3. 태그의 성격이 미정의
       - 태그가 상대방에게 보이면 메타게임 발생 ("상대 S가 낮으니 밀어붙이자")
       - 태그는 순수 디버깅/관찰용이어야 함

  ■ 해결 — v0.3 개정:
    1. 인식에서 시스템 메시지 언급 완전 제거 (Advocate, Skeptic 양쪽)
       - "시스템 메시지는 토의 방향의 이탈을 교정하기 위한 장치이다." 삭제
       - D-H의 "시스템 메시지 교정이 개입할 수 있다" 삭제
    2. 태그 속성 명시 (Advocate O-1-4, Skeptic O-1-5 신설):
       - "태그는 디버깅용이며 상대방에게 전달되지 않는 휘발성 데이터이다."
       - "태그는 나의 내부 상태를 기록할 뿐, 토의 흐름에 영향을 주지 않는다."
    3. Observer 구조 전면 개편:
       - Temp+/Temp- 적응적 재실행 시스템 [폐기]
       - 고정 섹션 구조 도입:
         · 1섹션 = 3회 주고받음 (6턴)
         · 기본 3섹션 운영
         · 섹션 1·2 결론 동일 → 섹션 3 스킵
       - O-3-0 섹션 구조 신설
       - O-3-4 섹션 간 비교 규칙 신설 ("동일 결론" 판정 기준 3가지)
       - O-4-1 "런 횟수" → "섹션 수" 변경
       - O-4-2 섹션 간 비교 보고로 변경

  ■ 교훈:
    - 시스템 개입을 없애면 블랙박스가 완성된다 — 평가자/비평자는 토의 상대만 인식
    - 적응적 복잡성(Temp 조정)보다 고정 구조(3섹션)가 더 안정적
    - 태그 = 휘발성 디버깅 데이터라는 정의가 메타게임 방지의 핵심
    - "개입하지 않는 관찰"이 진정한 블랙박스 — 관찰자는 섹션 사이에만 판정

[12] 데이터 필터 소스 신뢰도 계층 확정 — 2026-03-14 세션 #3
────────────────────────────────────────

  ■ 증상:
    - [11]에서 data-filter 에이전트가 research_queue와 Plans를 포함하여 비평자에게 전달하는 구조
    - research_queue = [구상] 수준 미검증 연구과제 → 비평자가 이걸 근거로 쓰면 안 됨
    - Plans = 미확정 계획 포함 가능 → 직접 논거로 쓰면 신뢰도 문제

  ■ 원인:
    - 비평자의 핵심 원칙: "검증된 과거 기록으로 반론한다"
    - research_queue는 검증의 반대편 — 미검증 아이디어 모음
    - 비평자가 미검증 데이터를 무기로 쓰면 C 태그(근거 수준) 전체의 신뢰가 무너짐
    - Plans는 "기존 계획과의 모순 지적"에는 유용하지만, 계획 자체가 확정 아닐 수 있음

  ■ 해결 — 데이터 소스 신뢰도 3계층:
    1. [높은 신뢰도 — 주력 소스]:
       - MasterLog — 프로젝트 의사결정 기록
       - True_Log — 검증된 사실 (최고 신뢰도)
    2. [낮은 신뢰도 — 참고용]:
       - Plans — 대화 흐름 참고용으로만 사용
       - Plans 출처 근거는 C-4 이하로 태깅 필수
       - data-filter가 Plans 항목에 "LOW RELIABILITY — reference only" 표기
    3. [제외 — 전달 금지]:
       - research_queue — 미검증 연구과제, data-filter가 읽지도 않음

  ■ 수정 파일:
    - agents/data-filter.md: O-1-2 데이터 소스 → 3계층 구조로 재구성
    - agents/skeptic.md: O-1-2 내부 데이터 → 높은/낮은 신뢰도 분리, Plans C-4 이하 제한
    - Prompts/Skeptic.md v0.5: 한글 원본 동기화
    - Prompts/DataFilter.md v0.1: 한글 원본 동기화
    - skills/sandbox-orchestrator/SKILL.md: data-filter 호출 시 research_queue 제외 명시

  ■ 교훈:
    - 비평자의 무기 품질 = 토의 전체의 신뢰도. 미검증 데이터가 들어가면 토의가 오염
    - "전부 전달하고 알아서 쓰게 하자"는 위험 — 에이전트는 받은 데이터를 무기로 쓰려 함
    - 데이터 접근 자체를 아키텍처 레벨에서 차단하는 것이 프롬프트 지시보다 확실
